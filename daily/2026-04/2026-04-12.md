## 🔧 话题一：用 Agent 写 Test Case + OPC 循环找 Bug

**发起人**：Xiaolin Quan | **时间**：16:11

> "以前就是自己打bug, 现在我都不找了，直接说你给我写test case, loop起来写，里面带上opc"

Xiaolin 描述了自己测试方式的根本性转变：从"人手动找 bug"变成"让 AI 写 test case，OPC 循环执行"。核心工作流：告诉 agent 需求 → agent 生成 test case → loop 反复执行 → OPC 并行化。

🧠 **解读**：这是一个从"人找 bug"到"AI 找 bug"的范式转变。传统 QA 是人写用例、人跑测试、人报 bug。Xiaolin 的做法把前两步全交给了 agent。OPC 作为并行引擎嵌入 test 流程，意味着可以同时跑多组测试。对 Edge Mobile PM 来说，这个思路可以直接应用到回归测试上：让 AI 生成 regression test case → loop 执行 → OPC 并行化，大幅提高测试覆盖率。
