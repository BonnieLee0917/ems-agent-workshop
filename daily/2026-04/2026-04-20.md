## ⚡ 话题一：Opus 4.7 确认可 1M + xhigh effort 支持

**发起人**：Jacky Zeng, Menci, Yue Liu | **时间**：00:02 - 02:05

承接 4/19 晚的讨论，Menci 凌晨实测确认 **Opus 4.7 可以 1M context**。

**技术细节**：

* **Jacky Zeng** 00:02 分享了一个 PR fix：[Fix Claude CLI reasoning effort fallback](https://github.com/Menci/copilot-gateway/pull/6)。原 PR 解析错误信息后每个请求多一个 RTT，Jacky 改成直接支持 xhigh
* **Menci** 01:32 质疑："这不是客户端的问题吗？用到 200k 以上不会爆？"
* **Jacky** 01:33："我试了没爆"。改了之后可以支持 xhigh effort
* **Menci** 02:02 **确认**："我确认了可以 1m！可喜可贺！"

![Menci 确认 1M 截图](images/2026-04-20-001.png)

* **Menci** 02:05 追问："能讲一下是怎么支持 xhigh 的吗？上面那个 PR 解析了半天错误信息，还在每个请求上白白增加了一个 RTT，非常无语"

🧠 **解读**：1M context 的关键不在模型端（模型本身支持），而在 gateway 和客户端的配置。Menci 的 copilot-gateway 需要正确处理 effort 参数，否则会被服务端拒绝降回 200k。Jacky 的 fix 是直接支持 xhigh 而不是 fallback 重试。这对所有用 copilot-gateway 代理的人都有价值。
