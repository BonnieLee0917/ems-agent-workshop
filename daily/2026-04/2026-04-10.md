## 📌 话题一：Weipeng Li & Shaobo Yan 入群作业（🔥 重点）

**发起人：** Weipeng Li, Shaobo Yan  
**时间：** 上午

两位新成员提交了 Jingxia Xing 的命题作业："分享过去三个月让你最震惊的 Agent 相关体验"。

### Weipeng Li（Theo Wang 补交作业）

**最震惊的：** Agent 工程的成熟度已到临界点。

核心观点：
- **Harness engineering** 让 Agent 不再是一次性对话，而是可以被调度、监控、重启的长期运行进程
- **记忆系统** 让 Agent 能跨任务积累经验，第 100 个 case 比第 1 个做得更快更好
- **多 Agent 编排** 让复杂任务被拆解成流水线，每个 Agent 专注自己的环节
- **验证闭环** — Agent 自己知道做没做对，不需要人盯着

**汗毛竖起来的 moment：** 即便模型能力从今天开始冻结，光靠工程手段，Agent 已经足以像"数字员工"一样无人值守地完成任务。人类的主要工作将切换为**规则制定、任务抽象、流程编排**，而不是具体执行。

**正在做的事：** 搭了一个 Agent 调度平台，让多个 Claude Agent 并行处理 Edge 浏览器 bug 修复，全链路无人介入。**两周自主修复 150+ 个 case，提交合入 90+ PR**。追踪指标叫 **Lights-out rate**（有多少比例的代码任务是 Agent 完全自主完成的，人类只在 Review 环节介入），目标是构建软件工程领域的"黑灯工厂"。

### Shaobo Yan

**最震惊的：** Claude Code 在复杂 task + 大型 codebase 场景下的长 loop 稳定性。

核心观点：
- CC 能在只设定目标和验收标准后，**几乎无打断地持续工作到结果产出**
- **Skill + Memory 系统** 是让 agent "长期成长"的关键设计。Skill 是对 BKM（Best Known Method）的固化；Memory 提供共性的解决思路和踩坑情况

**汗毛竖起来的 moment：** Agent 的**主动性和并行化**。在 auto pump 实验中，CC 不会机械按步骤执行，而是**主动判断可并行的步骤并同时推进**，并且会向你解释为什么这样做更高效。

**接下来要做的：** 在 auto pump 中拆解职责边界，引入 agent team：
- **Pump UI Monitor** — 负责验证、ownership take、任务分发
- **Pump Fixer** — 专注问题调查和修复
- 支持本地和多机器协同

## 📌 话题四：GPT 5.4 vs Opus — 知识库深度挖掘

**发起人：** Lingyan Zhao, Jingxia Xing  
**时间：** 上午

- **Lingyan Zhao** 对比发现 **GPT 5.4 对 concept 的延展能力比 Opus 更好**
- 她把知识库丢给 Opus 后，Opus 写的整理 plan 没有提出需要抽象出一层 concept，是 GPT 5.4 提出的
- **Jingxia Xing** 表示怀疑："真的有这么大区别么？"
- **Lingyan Zhao**："有，我的知识库需要深度挖掘"

**🧠 解读：** 这说明不同模型在不同任务类型上确实有差异。Opus 擅长代码和长 loop，GPT 5.4 在抽象概念提炼和知识整理上可能更强。根据任务类型选模型是有道理的。
