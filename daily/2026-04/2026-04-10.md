**发起人**：Xiaolin Quan, Jingxia Xing
**时间**：15:05 - 15:30

群里掀起一股"刷 token 排行榜"的热潮：

2. **Jingxia Xing** 透露正在开发 OPC 0.7 版本，核心逻辑是 `loop opc stop-hook`（循环执行 + 停止钩子），解决长任务中断问题。
3. **趣事**：Jingxia Xing 把自己的 Claude Code 用挂了。AI 陷入无限循环，不停说"Let me invoke the oop skill"然后又打字停不下来。原文非常搞笑。
4. 群里讨论"想一天刷 1B token 还是很有难度的"，有人调侃"到 2B 的时候，你就能涌现了"。
5. Dale Xiao 提出 "managed agents 是新中台"的观点，对应 Jingxia Xing 的"不需要中台了，给一个 plugin，你的 agent 去消费就 okay 了"。

### 🧠 解读 & 应用建议

OPC 本质上是一个**让 Claude Code 并行执行多个任务**的工具框架。"刷 token"不是目的，而是副产品。真正有价值的是：并行化 = 吞吐量 = 单位时间产出。

对 Edge Mobile PM 的启示：
- **Agent 可靠性是个真问题** — Jingxia 的 CC 挂了就是例子。长任务需要 loop + stop-hook 机制来保证容错。这和我们的日报定时任务遇到的 LLM 超时问题本质一样。
- **"不需要中台，给 plugin 就行"** — 这个观点对产品架构有启发。如果未来 Edge 的功能模块都能以 plugin 形式暴露给 agent，用户就能用自然语言组合功能，而不是在 UI 里翻菜单。

## 🔧 话题四：OpenClaw + pew Token 统计修复

**发起人**：Tao Li
**时间**：17:14

### 核心内容

Tao Li 发现并修复了一个技术问题：

**问题**：OpenClaw 通过 copilot-proxy（LiteLLM 本地代理）转发请求时，token 用量全部显示为 0，导致 [pew](https://pew.nocoo.xyz) 无法统计。

**根因**：OpenClaw 把 `localhost` 端点归类为 "local endpoint"，默认禁用了 `supportsUsageInStreaming`，导致请求不携带 `stream_options: { include_usage: true }`。

**修复**：在 `~/.openclaw/openclaw.json` 中给 copilot-proxy 下的每个 model 加上 compat 配置：

```json
{
  "id": "opus",
  "api": "openai-completions",
  "compat": {
    "supportsUsageInStreaming": true
  }
}
```

修改后重启 gateway 即可。注意：历史会话数据无法恢复。

### 🧠 解读 & 应用建议

这是一个非常实用的技术细节。如果你在用 OpenClaw + pew 统计 token 消耗，而发现数据全是 0，大概率就是这个问题。

对 Edge Mobile PM 的启示：
- 这说明 **数据统计的完整性需要主动验证**。我们做产品的 telemetry 也一样。如果某个指标突然降为 0，不一定是用户不用了，可能是埋点配置出了问题。
- pew 这个工具（token 用量可视化）值得关注。如果团队要考核 AI 工具使用效率，pew 可以作为一个参考。
